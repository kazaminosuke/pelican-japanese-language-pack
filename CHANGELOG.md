# Changelog

このファイルには各リリースの変更点を日本語でまとめます。

**運用ルール**: バージョンを上げてリリースする際は、`[未リリース]` セクションの
内容を `## [x.y.z] - YYYY-MM-DD` として確定し、先頭に新しい空の `[未リリース]`
セクションを追加してください。GitHub Actions のリリースワークフローはこの
ファイルから該当バージョンのセクションを抽出し、GitHub Release の本文として
そのまま使用します。
